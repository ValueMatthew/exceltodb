## 背景与问题

在“选择表”步骤中，不论用户通过手动校验选择目标表，还是通过系统推荐选择目标表，只要匹配度 ≥ 90% 后即可进入“导入设置”。但用户仍缺少一个“最后核对”的能力：快速查看目标表的字段结构与目标表中的真实示例数据（5 行），以确认选表无误。

本设计新增一个“预览目标表（字段 + 5行数据）”的入口，通过弹窗展示字段与示例数据，降低误导入风险。

## 目标与非目标

### 目标

- 在“选择表”页面中，当目标表匹配度 ≥ 90% 后，提供一个按钮入口：**预览目标表（字段 + 5行数据）**。
- 弹窗展示：
  - 表名
  - 匹配度 \(xx%\) 以及阈值 \(90%\)（匹配度直接使用当前页面已有结果，不重复计算）
  - 目标表**全部字段**
  - 目标表真实示例数据 **5 行**（不排序，直接 `LIMIT 5`）
- 当目标表无数据（0 行）时：
  - 仍展示字段列表
  - 数据区提示“该表暂无数据”

### 非目标

- 不做脱敏、字段白名单、权限控制（后续如需要可另起需求）
- 不做排序规则（明确不排序，避免额外依赖主键/时间字段）
- 不在预览接口中重新计算匹配度（避免重复计算与前端显示不一致）

## 现状概览

- 前端“选择表”页面：`client/src/components/TableRecommend.vue`
  - 已支持：手动指定目标表 + 校验（`/api/validate-table`）与系统推荐表（`/api/recommend`）
  - 已有：当选表成功后出现“导入设置”区域（导入模式、冲突策略、开始导入）
- 后端已支持：
  - `GET /api/tables/{databaseId}/names`
  - `POST /api/validate-table`
  - `POST /api/recommend`

## 方案（已确认）

采用“新增专用预览接口 + 前端弹窗展示”的方案。

## 前端设计

### 入口位置与出现条件

- 入口位置：放在“导入设置”卡片内（与“开始导入”同一块区域）
- 出现条件：
  - 当前已有已选定目标表（即 `selectedTableInfo` 已存在）
  - 且当前页面已知匹配度 ≥ 90%
    - 手动模式：来自 `/api/validate-table` 返回的 `score/threshold`
    - 推荐模式：来自 `/api/recommend` 返回的 `recommendations[i].score`（选中的那一项）

按钮文案：`预览目标表(字段+5行)`

### 弹窗交互

- 点击按钮后弹出 `el-dialog`
- 弹窗头部信息：
  - 标题：`目标表预览：<tableName>`
  - 副标题：`匹配度 xx%（阈值 90%）`
    - 匹配度数值直接使用当前页面已有匹配度（不通过接口重新计算）
- 弹窗内容区：
  - 字段列表（全部字段）
  - 示例数据表格（最多 5 行）
  - 若 0 行：示例数据区显示“该表暂无数据”，字段列表仍正常展示

### 状态与错误提示

- 弹窗加载态：请求进行中展示 loading
- 请求失败：弹窗内提示“预览失败，请稍后重试”，并允许用户关闭弹窗

## 后端 API 设计

新增接口（只读预览）：

### GET `/api/table-preview`

**Query 参数**

- `databaseId`（必填）
- `tableName`（必填）
- `limit`（可选，默认 5；服务端强制上限 5）

**响应**

```json
{
  "tableName": "orders",
  "columns": ["id", "order_no", "amount"],
  "rows": [
    { "id": 1, "order_no": "A001", "amount": 12.34 },
    { "id": 2, "order_no": "A002", "amount": 56.78 }
  ]
}
```

**SQL 行为**

- 字段：取目标表全部列（与 `SELECT *` 返回一致）
- 示例数据：`SELECT * FROM \`<tableName>\` LIMIT 5`
- 不做排序（按业务确认的 C 选项）

**异常处理**

- 任何异常：HTTP 500（延续现有 controller 风格）

## 验收标准

- [ ] 当目标表匹配度 ≥ 90% 并进入“导入设置”后，展示按钮“预览目标表(字段+5行)”
- [ ] 点击按钮弹出弹窗，展示表名 + 匹配度（页面已有值）+ 阈值 90%
- [ ] 弹窗展示目标表全部字段
- [ ] 弹窗展示目标表真实示例数据最多 5 行（不排序）
- [ ] 当目标表 0 行数据时，字段仍展示，数据区提示“该表暂无数据”
- [ ] 请求失败时有明确提示并可关闭弹窗
