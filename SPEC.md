# Excel To MySQL 导入工具 - 产品规格说明书

## 1. 产品概述

### 1.1 产品简介
面向业务人员的Web应用，实现Excel数据一键导入MySQL数据库。

### 1.2 目标用户
- 业务人员（非技术背景）
- 需要频繁将Excel数据导入数据库

### 1.3 核心价值
- 无需编写SQL或了解数据库知识
- 智能推荐目标表，减少选择成本
- 支持自动建表，降低使用门槛

---

## 2. 功能规格

### 2.1 数据库选择
| 规格 | 说明 |
|------|------|
| 配置方式 | IT人员在服务端 config.yaml 中配置 |
| 用户操作 | 业务人员仅从下拉列表选择数据库 |
| 连接池 | HikariCP |

**交互流程**：
1. 用户从下拉列表选择目标数据库
2. 系统验证连接并显示连接成功
3. 用户点击"下一步"进入文件上传

### 2.2 文件上传
| 规格 | 说明 |
|------|------|
| 支持格式 | .xlsx、.xls、.csv |
| 文件大小限制 | 最大 100MB |
| Sheet支持 | 仅支持单个Sheet（暂不考虑多Sheet） |

**交互流程**：
1. 用户拖拽或点击上传文件
2. 系统解析文件并显示基本信息（文件名、Sheet名、行数）
3. 用户点击"下一步"进入数据预览

### 2.3 数据预览
| 规格 | 说明 |
|------|------|
| 预览范围 | 前100行数据 |
| 显示内容 | 列名列表、数据行预览 |

**交互流程**：
1. 系统展示前100行数据
2. 用户确认列名正确后点击"下一步"

### 2.4 智能表推荐
| 规格 | 说明 |
|------|------|
| 推荐数量 | TOP 1 |
| 匹配维度 | 表名相似度、列名匹配率、主键匹配加分 |
| 无匹配阈值 | 低于30%认为无合适表 |

**推荐算法**：
```
输入：Excel列名列表、MySQL所有表及其列名（含主键）

对每个表计算匹配分数：
  - 表名相似度（Excel文件名 vs 表名）
  - 列名匹配率（Excel列在表中存在的比例）
  - 主键匹配加分（表有主键且Excel中有对应列）

返TOP 1推荐，若最高分<30%则返回空
```

**交互流程**：
1. 系统读取MySQL元数据
2. 计算并显示推荐的表（含匹配分数）
3. 用户选择目标表或选择"创建新表"

### 2.5 自动建表（可选）
当无合适表时，用户可选择自动创建。

| 规格 | 说明 |
|------|------|
| 建表方式 | 系统根据Excel数据自动推断列类型 |
| 用户确认 | 建表SQL需用户确认后执行 |

**类型推断规则**：
| Excel数据类型 | MySQL类型 |
|--------------|-----------|
| 整数 | INT |
| 大整数 | BIGINT |
| 小数/金额 | DECIMAL(32,18) |
| 字符串(≤255) | VARCHAR(255) |
| 字符串(>255) | TEXT |
| 布尔值 | TINYINT(1) |
| 日期 | DATETIME |

### 2.6 导入策略
用户在导入前需选择导入模式：

| 模式 | 说明 | SQL行为 |
|------|------|---------|
| 清空导入 | 导入前清空原表数据 | TRUNCATE TABLE + INSERT |
| 增量导入 | 直接追加数据 | INSERT |

### 2.7 主键冲突策略
当目标表有主键时，用户选择冲突处理方式：

| 策略 | 说明 | SQL行为 |
|------|------|---------|
| 更新（UPSERT） | 冲突时更新现有记录 | INSERT ... ON DUPLICATE KEY UPDATE |
| 忽略 | 冲突时跳过该记录 | INSERT ... IGNORE |
| 报错（默认） | 冲突时终止导入并回滚 | INSERT（默认行为） |

**注意**：
- 选择"清空导入"模式时，主键冲突策略自动失效
- 界面需显示目标表是否有主键

### 2.8 数据导入
| 规格 | 说明 |
|------|------|
| 批处理大小 | 每批5000条 |
| 事务控制 | 整个导入过程使用事务包裹 |
| 错误处理 | 出错时全部回滚，返回具体错误原因 |
| 进度展示 | 前端显示导入进度条 |

## 3. 技术架构

### 3.1 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + Element Plus |
| 后端 | Java 17 + Spring Boot 3 |
| Excel解析 | Apache POI |
| CSV解析 | OpenCSV / Apache Commons CSV |
| MySQL客户端 | mysql-connector-java |
| 连接池 | HikariCP |

### 3.3 API设计
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/databases | 获取可用数据库列表 |
| POST | /api/upload | 上传Excel/CSV文件 |
| GET | /api/preview/{filename} | 获取预览数据 |
| GET | /api/tables/{databaseId} | 获取所有表及列信息（含主键） |
| POST | /api/recommend | 获取推荐表 |
| POST | /api/import | 执行数据导入 |
| POST | /api/create-table | 创建新表 |

### 3.4 /api/import 请求参数
```json
{
  "filename": "orders.xlsx",
  "databaseId": "prod_erp",
  "tableName": "orders",
  "importMode": "INCREMENTAL",
  "conflictStrategy": "UPDATE"
}
```

## 4. 验收标准

### 4.1 功能验收
- [ ] 支持上传 .xlsx、.xls、.csv 文件
- [ ] 数据预览正确显示前100行
- [ ] 数据库下拉列表正确显示 config.yaml 中配置的数据库
- [ ] 表推荐算法正确计算匹配分数并返回TOP 1
- [ ] 表推荐结果中包含主键信息
- [ ] 自动建表能正确推断列类型
- [ ] 清空导入模式：原表数据被清空，Excel数据全部导入
- [ ] 增量导入模式：原表数据保留，新数据追加
- [ ] UPSERT策略：主键重复时更新而非报错
- [ ] 忽略策略：冲突行被跳过而非全量回滚
- [ ] 报错策略：主键冲突时终止导入并回滚
- [ ] 导入进度条正确显示进度
- [ ] 导入完成后展示成功/失败结果

### 4.2 性能验收
- [ ] 10万行数据导入能在合理时间内完成
- [ ] 内存占用稳定，无内存泄漏

### 4.3 错误处理验收
- [ ] 文件格式错误时给出明确提示
- [ ] 数据库连接失败时给出明确提示
- [ ] 导入出错时回滚全部数据并显示错误原因
- [ ] 主键冲突时根据策略正确处理
